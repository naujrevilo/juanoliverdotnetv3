Las mejores prácticas de seguridad son procedimientos probados que ayudan a proteger sistemas y datos de manera efectiva.

## Gestión de Identidad y Acceso

### Principio de Menor Privilegio

Otorgar solo los permisos mínimos necesarios para realizar una tarea.

**Implementación:**

- Revisar permisos periódicamente
- Eliminar accesos de usuarios inactivos
- Usar grupos y roles en lugar de permisos individuales
- Documentar justificación de accesos privilegiados

### Autenticación Multifactor (MFA)

Requerir múltiples factores de verificación para acceder a sistemas críticos.

**Tipos de factores:**

- **Algo que sabes**: Contraseña o PIN
- **Algo que tienes**: Token, smartphone, tarjeta
- **Algo que eres**: Huella digital, reconocimiento facial

## Protección de Datos

### Cifrado en Reposo y en Tránsito

Proteger datos mediante cifrado tanto almacenados como durante transmisión.

```bash
# Cifrar disco completo con LUKS
cryptsetup luksFormat /dev/sda1
cryptsetup luksOpen /dev/sda1 encrypted_disk

# Forzar HTTPS en nginx
server {
    listen 443 ssl http2;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
}
```

### Clasificación de Datos

Categorizar información según sensibilidad y aplicar controles apropiados.

**Niveles:**

- **Público**: Sin restricciones
- **Interno**: Solo personal autorizado
- **Confidencial**: Acceso limitado con MFA
- **Restringido**: Máxima protección, auditoría completa

## Gestión de Vulnerabilidades

### Escaneo Regular

Identificar vulnerabilidades antes que los atacantes.

**Herramientas:**

- **Nessus**: Escáner de vulnerabilidades comercial
- **OpenVAS**: Alternativa open source
- **Qualys**: Solución basada en la nube

```bash
# Escaneo básico con OpenVAS
gvm-cli --gmp-username admin --gmp-password admin socket \
  --xml "<create_target><name>MiServidor</name><hosts>192.168.1.100</hosts></create_target>"
```

### Gestión de Parches

Aplicar actualizaciones de seguridad de forma oportuna.

**Proceso:**

1. Monitorear avisos de seguridad
2. Evaluar impacto y urgencia
3. Probar en entorno de desarrollo
4. Desplegar en producción con ventana de mantenimiento
5. Verificar funcionamiento post-parche

## Monitoreo y Respuesta

### Registro y Auditoría

Mantener logs detallados de actividades del sistema.

**Qué registrar:**

- Intentos de autenticación (exitosos y fallidos)
- Cambios de configuración
- Acceso a datos sensibles
- Actividad de usuarios privilegiados

### Sistema de Detección de Intrusiones (IDS)

Monitorear red y sistemas en busca de actividad maliciosa.

```bash
# Configurar Snort IDS
# /etc/snort/snort.conf
var HOME_NET 192.168.1.0/24
var EXTERNAL_NET !$HOME_NET

# Ejecutar Snort
snort -A console -q -c /etc/snort/snort.conf -i eth0
```

## Concienciación de Seguridad

### Capacitación Continua

Educar a usuarios sobre amenazas y buenas prácticas.

**Temas:**

- Identificación de phishing
- Gestión segura de contraseñas
- Protección de dispositivos móviles
- Reporte de incidentes

### Simulaciones de Phishing

Realizar pruebas controladas para medir efectividad de capacitación.

**Métricas:**

- Tasa de clics en enlaces maliciosos
- Reporte de correos sospechosos
- Tiempo de respuesta ante simulacros

## Copias de Seguridad

### Regla 3-2-1

Estrategia de respaldo efectiva.

- **3 copias** de datos (original + 2 respaldos)
- **2 medios diferentes** (disco, cinta, nube)
- **1 copia fuera del sitio** (protección contra desastres)

```bash
# Backup automatizado con rsync
rsync -avz --delete /data/ backup@remoto:/backups/$(date +%Y%m%d)/

# Verificar integridad con checksums
sha256sum /data/* > checksums.txt
```

## Conclusión

Implementar estas mejores prácticas reduce significativamente el riesgo de incidentes de seguridad y mejora la postura general de ciberseguridad de la organización.
